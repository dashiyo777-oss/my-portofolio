# 健康・生活お役立ち情報 評価ランキングアプリ 企画書

> ショート動画に溢れる「健康・生活の役立ち情報」を、**本当に効くか評価**し、
> **ジャンル別にランク付け**して分かりやすく届ける。海外の優良コンテンツも
> **発掘・要約・多言語化**してデータベース化し、効果を実感したユーザーから
> 貢献者へ**チップ（投げ銭）**が回る——信頼できる「生活知」のプラットフォーム。

- ステータス: コンセプト検討中
- 作成日: 2026-06-29
- 対象ブランチ: `claude/health-tips-rating-app-ogpbir`

## 1. 背景・課題

TikTok / YouTube ショート / 抖音 などには、健康・生活のお役立ち動画が大量にある。

- 「死ぬほど効く神ツボ」「1日20回で視力回復」「股関節つまりが30秒でスッキリ」など
- 再生数は数十万〜数百万と**需要は巨大**

一方で、ユーザー側にはこんな課題がある。

1. **玉石混交** — どれが本当に効くのか分からない。誇張・無根拠なものも多い。
2. **検索しづらい** — 「生活知」は体系化されておらず、目的から探せない。
3. **言語の壁** — 海外には日本未流入の優良ノウハウが大量にあるが届かない。
4. **信頼の欠如** — 医療・健康系は誤情報のリスクが高く、安心して試せない。

## 2. ソリューション（提供価値）

| 機能 | 内容 |
|---|---|
| **評価** | コンテンツが「本当に役立つか」を多層的に評価（後述 §4） |
| **ランク付け** | ジャンル別・目的別に分かりやすくランキング表示 |
| **ジャンル提供** | 肩こり/腰痛、睡眠、節約、家事、育児…など目的から探せる |
| **海外発掘＋多言語化** | 海外の高評価ノウハウを発掘し、自前解説として多言語提供 |
| **データベース化** | 「どの生活知が・どの国でバズり・効果評価はどうか」を資産化 |
| **チップ（投げ銭）** | 効果を実感したユーザーが貢献者へ後払いで感謝を還元 |

### コアの差別化（堀 / moat）
動画そのものではなく、**「評価・エビデンス・多言語解説のデータベース」**が本体価値。
これは競合に真似されにくい独自資産になる。

## 3. 海外コンテンツの取り込みと著作権対応

> 「動画を運ぶ」のではなく「**知見と評価を運ぶ**」設計にすれば合法的に成立する。

### 何がNGで、何がOKか

| やり方 | 可否 | 理由 |
|---|---|---|
| 他人の動画をDLして自アプリで再配信 | ❌ | 複製権・公衆送信権の侵害 |
| 動画を勝手に翻訳（字幕・吹替）して転載 | ❌ | 翻訳権・翻案権の侵害 |
| 元動画への**公式リンク／埋め込み**＋自前解説・評価 | ⭕ | 再配信に当たらない |
| 動画の**ノウハウ（事実・手法）だけ抽出**し自前で再構成 | ⭕ | 事実・アイデアに著作権はない（表現のみ保護） |
| 投稿者と**契約／許諾**して翻訳・掲載 | ⭕ | 最も安全。収益分配・チップと相性◎ |

### 採用する2モデル（ハイブリッド）

- **① ノウハウDBモデル（スケール重視）**
  動画自体は扱わず、公式埋め込み／出典リンクで参照。
  本体価値は「評価・ランク・エビデンス・自前の多言語解説」という二次情報。
  → 事実・手法に著作権が及ばない原則を活用。
- **② クリエイター提携モデル（信頼・収益重視）**
  優良投稿者と正式提携し、許諾を得て翻訳・掲載。チップ／収益分配を提携の動機にする。

→ **①でDBを作り、評価の高い投稿者を②で口説く**流れが最も筋が良い。

### 翻訳の扱い（重要）
- 動画の音声/字幕をそのまま翻訳して載せる → ❌ 翻訳権侵害
- 運営が内容を理解し、**自分の言葉で要約・解説したものを多言語化** → ⭕（自前の著作物）

→ 「翻訳」ではなく**「解説の多言語化」**に倒す。安全かつ価値も高い。

### その他の遵守事項
- **各プラットフォーム規約**: 著作権法とは別に、スクレイピング・自動収集を禁じる規約が多い。公式API／公式埋め込みの範囲で運用する。
- **健康表現の二重チェック**: 海外では合法でも日本の薬機法でNGな表現があり得る。取り込み時に規制フィルタ工程を通す（AIで補助可能）。

## 4. 「本当に役立つか」の評価設計

ユーザー投票だけだと「バズった動画＝高評価」になりがち。主観・プラシーボも混ざる。
そこで**多層評価**にする。

1. **体感評価** — 一般ユーザーが「効いた／効かなかった」を星・継続日数で報告
2. **専門家監修** — 医師・専門家による監修バッジ（特に医療系は必須）
3. **エビデンスタグ** — 科学的根拠の有無・出典の質をタグ化（AIで一次照合を補助）
4. **継続効果** — 一定期間後の追跡で「短期だけ／継続的に効く」を区別

→ 表示は「**体感レビュー**」と「**科学的根拠**」を明確に分離し、誤認を防ぐ。

## 5. 収益モデル

- **チップ／投げ銭** — 効果実感後の後払い。納得感が高く差別化に。
- **プレミアム閲覧** — 専門家監修・エビデンス付きの上位コンテンツ
- **クリエイター提携の収益分配**
- （将来）健康関連の認証広告・アフィリエイト（規制順守前提）

> 注意: 少額投げ銭は Apple / Google 課金手数料（〜30%）の影響大。
> 海外展開時は**送金ライセンス・税務・各国決済**が重いので段階導入する。

## 6. リスクと対策

| リスク | 対策 |
|---|---|
| **医療・健康情報の法規制**（薬機法／景表法／FTC・FDA・医療広告規制） | 表現を「体感」と「根拠」に分離／専門家監修必須／効能断定を避ける |
| アプリ側のランク付け＝お墨付き責任 | 評価軸を「エビデンス・監修の有無」に寄せ、断定を避ける |
| 著作権・転載 | §3の「知見と評価を運ぶ」設計を徹底 |
| 評価の信頼性（バズ＝高評価問題） | §4の多層評価 |
| 投げ銭の送金・税務（海外） | 1国で固めてから段階展開 |

## 7. MVP（最小実装）— まず絞って検証

最初から全部やらない。**1ジャンル × 1国**で検証する。

| 項目 | MVP の方針 |
|---|---|
| ジャンル | 「肩こり・腰痛などセルフケア」1本に絞る |
| 評価 | ①体感★ ②専門家監修バッジ ③エビデンス有無タグ |
| コンテンツ | 公式埋め込み＋自前の要約・解説（再配信しない） |
| 収益 | まずチップではなく**監修付きプレミアム閲覧**で需要検証 |
| 地域 | 日本のみ（規制対応を1国で固める） |

## 8. 海外展開ロードマップ（段階）

1. **国内MVP** — 1ジャンルで評価・DB・UXを磨く
2. **海外発掘の自動化** — 公式APIで高評価ノウハウを収集 → 規制フィルタ → 多言語解説化
3. **多言語展開** — 解説の多言語化でユーザー基盤を海外へ
4. **チップ／送金の段階導入** — 決済・税務・ライセンスを国ごとに整備
5. **クリエイター提携** — 各国の優良投稿者と提携し収益分配

## 9. 総評

- コンセプトは**「あり」かつ今っぽい**。需要は実証済み（高い再生数）。
- 成否は **「健康情報の信頼性をどう担保し、法的リスクをどう回避するか」** にほぼ集約。
- ここを設計の中心に据えれば、単なる動画まとめではなく
  **「信頼できる生活知のランキング・データベース」**という強いポジションが取れる。
