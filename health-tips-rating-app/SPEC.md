# 効く（Kiku）— 技術・設計ハンドオフ仕様（MVP）

> 健康・生活のお役立ち情報を「**体感・科学的根拠・専門家監修**」で評価し、
> ジャンル別に**ランク付け**して届けるアプリ。海外コンテンツの発掘・多言語化と、
> 効果を実感したユーザーから貢献者への**チップ**まで見据える。
>
> 事業コンセプトは `CONCEPT.md` を正典とし、本書は**実装・データ・設計の手戻りを防ぐ**ための技術仕様。

- ステータス: **MVP 実装済み。`index.html` を開くだけで動作。**
- 方針: 既存ポートフォリオと同じく**追加依存なし・ビルド工程なしの素のWeb技術**。

---

## 0. なぜこの設計か（手戻り防止の3原則）

1. **データ＝将来のDBスキーマ**：`data.js`（`window.APP_DATA`）の形を、そのまま
   バックエンドのテーブルに写せるようにした。i18n は最初から `{ ja, en }` 構造。
   → サーバ化・多言語追加でカラム追加だけで済み、作り直しが起きない。
2. **コンセプトの核をコードに固定**：「バズ＝高評価」を防ぐ**透明なランキングスコア**を実装（§4）。
3. **薬機法配慮をUIに内蔵**：体感と科学的根拠を**画面上で必ず分離**し、効能を断定しない。
   注意書き・免責を全画面に常設。→ 規制対応を後付けで作り直さない。

## 2. データモデル（= 将来のDBスキーマ）

### genres
`id, name{ja,en}, icon, color`

### contributors（チップの受け手・海外含む）
`id, name, origin, color, expert?, bio{ja,en}`

### contents（知見＋評価。**動画そのものは持たない**）
| フィールド | 内容 |
|---|---|
| `id, genreId, contributorId, origin` | 関連・発掘元の国 |
| `title{ja,en}` / `summary{ja,en}` | **運営が自分の言葉で書いた**見出し・要約（自前著作物） |
| `steps[ {ja,en} ]` | 手順（事実・手法。著作権の及ばない情報） |
| `caution{ja,en}` | 注意・禁忌（薬機法配慮） |
| `source{ type,label,url }` | **公式リンク/埋め込みのみ**（再配信しない / CONCEPT.md §3） |
| `evidence{ level, note{ja,en} }` | 科学的根拠: `none / anecdotal / expert / study` |
| `review{ status, by }` | 専門家監修: `unreviewed / expert` |
| `minutes, tags[]` | 目安時間・タグ |
| `stats{ tried, resonated, notHelpful }` | 体感の集計（シード値） |

> 著作権の肝：contents は「動画を運ぶ」のではなく「**知見と評価を運ぶ**」。
> summary/steps は自前、出典は公式リンクのみ。これで合法的に海外発掘を取り込める。

## 3. 評価の二層分離（薬機法配慮）

画面では必ず分けて表示し、効能を断定しない。

- **体感レビュー**（`stats`）… 「効いた / 自分には今ひとつ」の主観。`feltRate` として%表示。
- **科学的根拠**（`evidence`）… 客観の度合い。`note` で限界も明示（例：「視力が戻る根拠はない」）。
- **専門家監修**（`review`）… 監修の有無。

全画面に免責（「本アプリは医療行為ではありません」）を常設。

## 4. ★ランキングの核：効くスコア（CONCEPT.md §4 をコード化）

```
score = 100 * ( 0.55*W + 0.25*E + 0.15*S + 0.05*P )

W : 体感「効いた」率の Wilson 下限（少数票に強い・荒れにくい）
E : 科学的根拠の重み  none .10 / anecdotal .35 / expert .70 / study 1.0
S : 専門家監修の重み  unreviewed .30 / expert 1.0
P : 試した人数の log 減衰 = min(1, log10(tried+1)/4)   ← 重みは僅か 0.05
```

**ねらい**：再生数（バズ）と「本当に効く」を分離する。人気(P)の重みを意図的に小さくし、
体感率・根拠・監修が低ければ、いくら試行数が多くても上位に来ない。

**実証（MVPデータ）**：最も試行数の多い「股関節つまり10回」（tried 9,800 / 体感54% / 根拠 anecdotal / 未監修）は
**最下位（score 47）**に沈み、上位は研究裏付け＋監修済みの項目が占める。
→ コンセプト「真に役立つ情報を上げる」がコードで担保されている。

ユーザー自身の評価（`localStorage`）はシード集計に合算され、スコアに即反映される。

## 5. 状態管理（端末内 → サーバ集計への布石）

`localStorage: kiku.state.v1`
```json
{ "lang": "ja",
  "user": { "<contentId>": { "tried": true, "feedback": "resonated|not_now|null", "tipped": 0 } } }
```
- `feedback` は体感集計（`resonated/notHelpful`）へ合算 → スコアに反映。
- この個人データの形を、将来そのまま匿名・同意ベースのサーバ集計 API に拡張できる。

## 6. 画面（MVP）

1. **ホーム/ランキング**：検索ボックス、ジャンルchip、**タグchip（横断）**、効くスコア順リスト
   （メダル・スコア・体感率・根拠/監修ピル）、ヒット件数。
2. **詳細**：スコア内訳（体感/根拠/監修を分離）、**クリック可能なタグ**、やり方、注意、出典（公式リンク）、
   貢献者＋チップ（mock）、「やってみた」→体感フィードバック。
- ハッシュルーティング：`#/`（全件）, `#/g/<genre>`（ジャンル）, `#/t/<tag>`（タグ横断・共有可）, `#/c/<id>`（詳細）。
- **検索＋タグ／ジャンルは組み合わせ可能**（検索語はハッシュに載せず、ジャンル/タグに重ねる）。
  検索はタイトル・要約・タグを日英横断でマッチ。入力中はリストのみ再描画しフォーカスを保持。
- 日英トグル（`navigator.language` 初期判定、`localStorage` 保存）。

## 9. 海外展開メモ

- i18n はデータ構造で `{ja,en}`。中・西・仏などは同じ仕組みでキー追加のみ。
- `contents.origin` と貢献者 `origin` で「発掘元の国」を可視化＝海外発掘の価値を表現。
- 規制は国ごとに異なる（薬機法/景表法/FTC・FDA等）。`evidence`/`review` を評価軸の中心に
  据えることで、断定を避けつつ国際展開の土台になる。
