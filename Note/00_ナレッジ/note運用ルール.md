# note 運用ルール（有料 / 無料の切り分け）

このプロジェクトの「記事（有料部分）」と「レター（無料部分）」の設計基準をここに固定する。
迷ったら必ずこのルールに立ち返る。

---

## 1. 無料パート（レター）と有料パートの役割

| | 無料パート（`04_レター_無料部分/`） | 有料パート（`03_記事_有料部分/`） |
| --- | --- | --- |
| 目的 | 信頼獲得と「続きを読みたい」動機づけ | 具体的な手順・数字・テンプレなど"実際に使える価値" |
| 出す情報 | 問題提起・結論の一部・実績の匂わせ・全体像 | 再現可能なノウハウ・具体例・失敗回避策・ツール設定 |
| 書き方 | フック（タイトル9割）→ 共感 → 結論の入口で有料ラインへ | 逆ピラミッドで即・核心。読んで手が動く粒度 |
| NG | 無料部分だけで満足させて終わる | 無料で言ったことの焼き直しだけ |

> 原則：**無料で「なるほど」、有料で「明日から使える」**。
> 無料パートは"予告編"であって"総集編"ではない。

## 2. 有料ラインの引き方

- 無料パートの最後は、**結論の「入口」または「最も知りたい問いを立てた直後」**で切る。
- 「ここから先は、私が実際に月◯万円を得たときの具体的な手順です」のように、
  有料で得られる便益を1文で明示してから切る。
- ただし煽りすぎない。CLAUDE.md の倫理（嘘を売らない）を最優先。

## 3. 価格・構成の目安（初期方針・実績に応じて更新）

- 単発記事：無料パート 1,500〜2,500字 ＋ 有料パート 2,000〜4,000字
- 価格：まずは低め（例 300〜980円）で実績と信頼を積む → 実績が出たら見直す
- マガジン化：テーマが溜まったらマガジンにまとめて継続収益へ

## 4. タイトル・タグ戦略

- タイトルは**最後ではなく最初**に仮決め（記事のコア＝最も鋭利な刃）。
- パワーワードは誠実さを保った範囲で（例「手取り」「実際に」「◯万円」「現実」）。
- タグは検索流入を意識して 3〜5個（例：#AI副業 #ClaudeCode #note販売 #在宅ワーク）。

## 5. 公開後のルーティン

1. note公開版を `05_公開済み/` にバックアップ（ファイル名に公開日）
2. `07_運用ログ/投稿カレンダー.md` に URL・公開日・価格を記録
3. 数日後、閲覧数・購入数・コメントを同ログに追記し、次記事の改善に反映
