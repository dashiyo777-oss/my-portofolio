# Note — AI副業リサーチ＆記事販売ワークフロー

このフォルダは、**AI副業に関する情報をリサーチ → 記事（有料部分）を執筆 → レター（無料部分）を作成 → note で販売**するための作業スペースです。

執筆はすべて、リポジトリ直下の [`CLAUDE.md`](../CLAUDE.md) と
[`knowledge/professional-writing-blueprint.md`](../knowledge/professional-writing-blueprint.md)
の「型」と「倫理」に従って行います。

---

## フォルダ構成と役割

| フォルダ | 役割 | 中身 |
| --- | --- | --- |
| `00_ナレッジ/` | 判断基準の置き場 | note運用ルール（有料/無料の切り分け・価格・タグ）、ライティング指針への参照 |
| `01_リサーチ/` | 一次情報の収集 | リサーチメモ（テンプレ）、ネタ帳、ソース一覧（出典・エビデンス管理） |
| `02_下書き/` | 執筆中の記事 | 記事下書きテンプレを複製して執筆 |
| `03_記事_有料部分/` | 完成した有料本文 | note の「ここから先は有料」以降に載せる中核コンテンツ |
| `04_レター_無料部分/` | 無料の導入・レター | 記事冒頭に置く無料パート（フック＋購入動機づけ） |
| `05_公開済み/` | 公開後アーカイブ | note公開版のバックアップ（URL・公開日を記録） |
| `06_素材/` | 図版・画像 | アイキャッチ、図解、スクショなど |
| `07_運用ログ/` | 運用・改善 | 投稿カレンダー、売上・反応・改善メモ |

---

## 基本ワークフロー（1記事の流れ）

1. **ネタ出し** … `01_リサーチ/ネタ帳.md` に問い・仮説をストック
2. **リサーチ** … `01_リサーチ/_テンプレート_リサーチメモ.md` を複製し、X・公的データ・一次情報を収集。出典は必ず `ソース一覧.md` に記録
3. **設計** … タイトル案と「1記事1テーマ」「起承転結」を先に決める（`02_下書き` のテンプレ冒頭）
4. **執筆** … `02_下書き/` で本文を書く
5. **分割** … 無料パート → `04_レター_無料部分/`、有料パート → `03_記事_有料部分/`
6. **チェック** … 執筆前チェックリスト（下記）を全項目クリア
7. **公開** … note に投稿し、`05_公開済み/` にバックアップ＋`07_運用ログ/投稿カレンダー.md` を更新

---

## ⚠️ 倫理の最優先事項（CLAUDE.md より）

- **嘘を売らない** … ビュー数や売上のために事実を曲げない
- **こたつ記事の排除** … ネット情報の切り貼りにしない。一次情報・裏取りが命
- **断定の回避** … 100%証明できないことは「〜の疑いがある」「〜と指摘されている」
- **出処の明示** … データの出典を必ず開示

## 執筆前チェックリスト（公開前に必ず確認）

- [ ] 一次情報・裏取りに基づいているか（憶測・こたつ記事でないか）
- [ ] タイトルは記事のコアを射抜く「最も鋭利な刃」か
- [ ] 1記事1テーマに絞れているか／捨てる要素を捨てたか
- [ ] 起で置いたキーワードを結で回収できているか
- [ ] 数字は核心部にピンポイント配置か（乱れ打ちしていないか）
- [ ] 断定できない情報は抑制表現・会話文で扱っているか
- [ ] 出典を明示したか
- [ ] 無料パートだけで「読む価値」と「続きを読みたい動機」が伝わるか
