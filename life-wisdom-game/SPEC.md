## 1. コンセプト

一言で：**「迷ったとき、世界中の偉人があなたの相談相手になってくれる人生シミュレーション。」**

- プレイヤーは自分の人生を歩み、**仕事・人間関係・恋愛・学業・お金**などの岐路で悩む。
- その瞬間に**世界の**偉人・有名人・成功者（孔子、スティーブ・ジョブズ、マザー・テレサ、坂本龍馬、アインシュタイン、ネルソン・マンデラ、オードリー・ヘプバーン等）が登場し、**励まし・叱咤激励・いたわり**の言葉をくれる。
- 「正解の決断」ではなく、**自分に響いた言葉を選ぶ**体験が中心。ゲームオーバーはなく、何度でも違う偉人に出会える温かい設計。

### 体験の狙い（プレイヤーに残るもの）
- その日・その悩みに効く一言を持ち帰れる（実用的な癒し）。
- 自分だけの「人生の物語」と「名言コレクション（お守り帳）」が育つ。

### ★コアバリュー（本作の最大の価値）
**「世界中の偉人・有名人・成功者の言葉を、悩みの文脈で引ける高品質データベース」そのものが資産。**

- 日本人に閉じず、東西・古今・分野（思想・科学・芸術・スポーツ・実業・人道）を横断する。
- 単なる名言集ではなく、「**どんな悩み・心の状態に効くか**」でタグ付けされている点が差別化。一般的な名言アプリは"出典別"、本作は"悩み別×心の状態別"で引ける。
- 可能なら**生の声（本人の実音声 / 一次資料の録音）**を添え、文字を超えた体験にする（→ §6.5 音声方針）。
- この**データの質・量・正確さ・権利のクリーンさ**が、課金に値する品質（→ §11）の核になる。

## 3. ゲーム設計の4つの軸

| 軸 | 内容 |
|---|---|
| 悩みカテゴリ | 仕事 / 人間関係 / 恋愛 / 学業 / お金 / 健康 / 自己実現 |
| 心の状態 | 落ち込み・舞い上がり・迷い・不安。状態で出る言葉が変わる |
| 偉人の個性 | 同じ悩みでもジョブズと孔子では答えが真逆 → 相性が生まれる |
| コレクション | 集めた名言を「お守り」として後で見返せる |

### 3.1 ステータス（人生パラメータ）
0〜100の連続値で保持。決断で増減する。

- `mind`（心のHP / メンタル）— 0でもゲームオーバーではなく「休息イベント」へ
- `wisdom`（叡智 / 経験値）— 名言を集める・熟考の選択で上昇
- `bonds`（絆 / 人間関係）
- `wealth`（お金）
- `passion`（情熱 / やる気）

UIには上位2〜3個のみ常時表示し、残りは詳細画面で見せる（情報過多を避ける）。

### 4.2 相性（任意機能・v1.5）
プレイヤーの選択履歴から「よく言葉を選んだ偉人」を“座右の偉人”として表示。リピート性を高める軽い仕掛け。

### 4.3 ★偉人ランクと解放（課金 × 経験のハイブリッド）
**設計の核：偉人には「ランク」があり、高ランクの偉人は課金では飛ばせず、経験（叡智 `wisdom`）を積んで解放する。** 課金は"偉人フェーズへの入口/パック"を開くもの、ランク解放は"プレイで稼ぐ"もの、と役割を分ける（→ pay-to-win 回避・継続性確保）。

**ランク階層（例）**

| rank | 称号 | 解放条件（経験ゲート） | 体験 |
|---|---|---|---|
| 1 | 入門の賢者 | 偉人フェーズ解放直後から | 平易で力強い励まし。間口を広く |
| 2 | 賢者 | `wisdom` 30〜 | 悩み特化の深い助言 |
| 3 | 大賢者 | `wisdom` 70〜＋特定カテゴリ実績 | 人生の核心を突く言葉・希少名言。**聖典の言葉**が現れ始める（§4.4） |
| 4 | 伝説の賢者 | `wisdom` 120〜＋座右の偉人実績等 | 看板級。生の声/特別演出を優先付与 |
| 5 | **神聖なる導き（極レア）** | `wisdom` 200〜＋複数カテゴリ達成等の最終条件 | **最高到達点。聖典＝聖なる叡智からの一節を、最も荘厳な演出で賜る**（§4.4） |

> rank 5 は"クリア後のご褒美"級の極レア。乱発せず、長い積み重ねの果てに出会う設計にする。

**ルール**
- `wisdom`（叡智）は名言を集める・熟考する選択で増える（§3.1）。**経験を積むほど、より高位の偉人に出会える。**
- **課金で高ランクを直接買うことはできない**（経験必須）。課金が解放するのは「偉人フェーズ／偉人パック（その人物が登場し得る状態）」まで。出会えるかは経験次第。
- 高ランク偉人は数を絞り"ご褒美感"を出す。登場時は特別演出（オーラ・専用SE・生の声優先）。
- データ：`rank`（1〜5）と `unlock`（`wisdom` 閾値や実績条件）を `sages.json` に持つ。`isPremium`/`packId` は課金境界（§10.2）。

### 4.4 ★聖典・宗教コンテンツの扱い（敬意と正確性ポリシー）
中盤以降、**聖書・コーラン（クルアーン）・仏典など聖典からの助言**が登場し、最高位 rank 5「神聖なる導き」へと至る。宗教は人の心の最も深い部分に関わるため、**正確性・敬意・中立性**を絶対条件とする。これを丁寧に守ることが、本作が"課金に値する品位"を持つ証になる。

**★確定方針：rank 5 は「聖典の言葉を賜る」形（神を一人称で演じない）。核は三大宗教（キリスト教/イスラム/仏教）。神道は扱わない。**

**A. 「神そのものを演じない」— 聖典の言葉を賜る形にする（★確定）**
- rank 5 を「神/アッラー/仏が一人称で喋るキャラクター」にはしない。代わりに **「聖典（聖なる叡智）の一節を、出典付きで授かる」** 表現にする。話者は神ではなく"聖なる言葉そのもの"。
  - 理由：神格の一人称演出は不敬・冒涜と受け取られやすく、ストア審査・国際展開でも高リスク。「聖典の言葉を引く」形なら敬意が保て、かつ"神レベルの重み"という狙いは達成できる。
- 預言者・開祖（イエス、ブッダ等）を扱う場合も、史的人物の教えとして敬意を持って引用。

**B. 正確性（事実性ポリシー §5.3 を宗教にも厳格適用）**
- 聖典の引用は**章節番号まで明記**（例：聖書「マタイ 11:28」、クルアーン「94:5-6」、法句経「Dhammapada 1」）。
- 誤った引用・誤帰属は厳禁。信頼できる版/定訳に基づき裏取り（`verified:true`）。意訳は `note` で明示。

**C. 中立・非布教（重要）**
- どの宗教も**優劣をつけない**。「正しい宗教」を説く・改宗を促す（布教）一切をしない。あくまで**普遍的な励まし・癒し・知恵**として提示する。
- **核は三大宗教（キリスト教/イスラム/仏教）**でバランスよく開始。ユダヤ教・ヒンドゥー・道教等は将来候補。特定信仰への誘導をしない。
- **神道は扱わない**（人生訓を説く聖典を持たず本枠の構造に合わないため、対象外とする）。

**D. 視覚表現の禁則（必読）**
- **イスラム：神（アッラー）および預言者ムハンマドを絵で描かない**（神学上の禁忌）。光・幾何学/アラベスク・カリグラフィー・荘厳な抽象演出で表す。
- 神格全般、原則として具象的な"顔"を与えず、**光・象徴・書（カリグラフィー）・自然**などの抽象表現にする。
- 聖典は装丁や象徴（光の書、蓮、十字など）で品位ある表現に。揶揄・パロディは一切しない。

**E. データ表現**
- 聖典系エントリは `sageId` を実在個人にせず、`"scripture_bible"` / `"scripture_quran"` / `"scripture_dhammapada"` 等の**聖典ID**で扱い、`source` に章節を必須化、`tradition` フィールドで伝統を持つ。
- rank 5 の "神聖なる導き" は、これら聖典エントリの中でも特に普遍的で穏やかな一節を厳選して割り当てる。

**F. 運用・レーティング**
- 宗教コンテンツは慎重な監修が望ましい（可能なら各伝統に明るい監修者のレビュー）。
- ストアの年齢・コンテンツレーティングに宗教表現の有無を正しく申告。地域による配慮が要る場合は出し分けを検討。

### 5.3 名言の事実性・権利ポリシー（最重要 / 課金品質の生命線）
**課金プロダクトでは「誤った引用」「権利侵害」が致命傷**になる。データベースの信頼性こそが商品価値（§1, §11）なので、以下を厳守する。

**事実性**
- `quote` は**実在偉人の実際の言葉（または定訳）**であること。原語を `quoteOriginal`、出典を `source`（演説・著作・年など）で必ず持つ。
- 出典が曖昧／創作の励ましは、`sageId` を実在人物にせず `"original"`（ナレーター／灯火の声）として扱い、混同を絶対に避ける。
- いわゆる「誤帰属の名言（misattributed quotes）」を排除するため、各 quote は出典の一次／信頼ソースで裏取りし、`verified: true` を立てたものだけを本番採用する。
- 翻訳引用は意訳である旨を `note` で補える構造にする。

**権利（テキスト）**
- 著作権保護期間内の人物の長い引用は**短い引用（引用法の範囲）**に留めるか、権利元の許諾を得る。古典・パブリックドメインを優先的に厚くする。

**権利（肖像・音声）→ §6.5 へ。**

### 5.4 セーブデータ（`localStorage`）
プレイヤーが**後から自分の歩みを辿れる**よう（§6.6 わが叡智の書）、各決断を「記録(record)」として時系列で残す。`journal` がその本体。

```json
{
  "stats": { "mind": 70, "wisdom": 25, "bonds": 50, "wealth": 40, "passion": 60 },
  "favoriteSage": "jobs",
  "turn": 12,
  "journal": [
    {
      "ts": "2026-06-15T09:12:00+09:00",
      "turn": 1,
      "eventId": "work_failure_01",
      "category": "work",
      "mood": "down",
      "title": "仕事で大失敗。もう辞めたい…",
      "chosenSageId": "jobs",
      "quote": "未来に向かって点と点を結ぶことはできない。振り返って初めて、点は線になる。",
      "source": "Stanford University Commencement Address, 2005",
      "mindBefore": 40,
      "mindAfter": 48,
      "playerNote": "",
      "feedback": "resonated"
    }
  ]
}
```

- `journal[]` … 経験した悩みと、その時受けた助言の記録。本作の"思い出"であり資産。
- `quote`/`source` はその時点の表示内容を**スナップショット保存**（後でデータが更新されても、当時受け取った言葉が変わらないように）。
- `playerNote` … 任意。プレイヤーが自分の言葉でメモを添えられる（自分の叡智化を促す）。
- `feedback` … 任意。その言葉が自分に響いたか（`"resonated"`/`"not_now"`/未回答）。§6.7 の「響き度」に使用。
- `favoriteSage` は `journal` の選択履歴から集計（最も言葉を選んだ偉人＝座右の偉人）。
- 旧 `collected`/`history` は `journal` に統合（重複保持しない）。

## 6. 画面仕様（MVP）

1. **タイトル画面**：作品名 / 「はじめる」「叡智コレクションを見る」。
2. **イベント画面**：上部にステータスバー、中央にイベント文、下部に偉人カード最大3枚（顔・名前・一言）。
3. **決断演出**：選んだ言葉が大きく表示 → ステータス変化アニメ → 「コレクションに加わった」トースト。
4. **わが叡智の書（叡智コレクション）画面**：→ §6.6。集めた言葉と歩みを辿る本作の心臓部。
5. **休息イベント**：`mind` が低いとき発生。決断なしで回復し、いたわりの言葉だけをもらえる。

### 6.6 ★「わが叡智の書」— 経験した悩みと、受けた言葉を後から辿る
ユーザー要望：**プレイヤーが経験した悩みと、その時受けた助言を後から辿れるように。本人の叡智として。**

本作のテーマ「集めた叡智が、やがて自分自身の叡智になる」を体現する中核画面。単なる名言一覧ではなく、**自分が何に悩み、どの言葉に救われてきたかという"人生の足跡"**を見返せる。

**2つの見え方（同じ `journal` データの切り口）**
1. **歩み(タイムライン)**：時系列で「いつ・どんな悩み・どの言葉を選んだか」を辿る。日記/巻物のように下へ流れる。当時の心の状態（`mindBefore→After`）も小さく示し、「あの時こう乗り越えた」が見える。
2. **叡智(コレクション)**：受け取った言葉を"お守り帳"として一覧。偉人・カテゴリ・心の状態でフィルタ。各カードに全文＋出典＋（あれば）生の声。

**仕掛け（"本人の叡智化"を促す）**
- 各記録に**自分のメモ（`playerNote`）**を添えられる。偉人の言葉＋自分の言葉で"我が叡智"になる。
- お気に入り（☆）を付けて「座右の言葉」に。タイトル/休息画面で時々そっと再提示。
- 振り返りの一言：節目で「あなたは○○の悩みを△回乗り越え、最も□□の言葉に支えられてきた」と要約。
- 共有（v2）：1記録を画像化して持ち出せる（§9）。**出典を必ず焼き込む**（誤帰属拡散の防止・§5.3）。

**設計メモ**
- データは `journal[]` のスナップショット（§5.4）。オフラインで完結、`localStorage` 永続。
- 同じ悩みを再経験しても別記録として積む（人生は繰り返し悩む、を肯定）。
- 件数が増えても軽いよう、一覧は遅延描画／カテゴリ別に折りたたみ。

### 6.7 ★言葉の「響き度」フィードバックと評価
ユーザー要望：言葉を受けて「助かった／参考にならなかった」等のフィードバックを集め、名言の評価を裏側（集計）で持つ（★評価のように）。

**狙い**：静的なDBを、反応で育つ"生きたDB"へ。「どの悩み×心の状態に、どの言葉が本当に効いたか」を蓄積し、(1) 推薦精度の向上、(2) 弱い名言の発見・差し替え、(3) パーソナライズ（その人に響く偉人へ寄せる）、(4) 社会的証明に活かす。正確さ（§5.3）に「効き目」の軸を足す。

**設計原則（ここが品質の肝）**
- **「名言の良し悪し」を採点しない**。聞くのは「**自分に・今・響いたか**（フィット度）」。偉人や言葉そのものを裁く形にしない。
- **演出を壊さない**。感情が動いた直後に星を強要しない。さりげない二択（例：「響いた♡」/「今はそうでもない」）を既定にし、星評価は任意・後追い（わが叡智の書 §6.6 で）。
- **聖典は特別扱い**（§4.4）。聖書・クルアーン等に星を付けさせない。「心に留めた」等の柔らかい反応のみ。神聖さを採点対象にしない。
- **オフライン優先 → 集計は後段**。まず端末内の個人評価で完結（プライバシー/サーバー不要）。匿名の全体集計はバックエンド導入の v2（§9）。

**段階導入**
- **Phase A（v1.5・サーバー不要）**：個人フィードバックを `journal[].feedback` に保存。用途＝お気に入り抽出、登場する偉人/言葉のパーソナライズ、振り返り。
- **Phase B（v2・バックエンド）**：匿名・同意ベースで集計し `quoteStats`（下記）を形成。用途＝推薦の重み付け、弱い名言の検出、「○%が"響いた"」の社会的証明。
- **配慮**：悩みはセンシティブ情報。集計は匿名化・最小化・明示同意。公開評価は荒らし対策（モデレーション/外れ値除去）。聖典は集計対象外か別指標。

**フィードバックのデータ形（個人 / 端末内）** … `journal[]` に付加（§5.4）
```json
{ "feedback": "resonated" }   // "resonated" | "not_now" | null（未回答）
```

**評価集計の形（裏側 / v2 バックエンド・匿名）**
```json
{
  "quoteKey": "work_failure_01:jobs",
  "byMood": { "down": { "resonated": 812, "not_now": 96 } },
  "resonateRate": 0.89,
  "samples": 908,
  "excludeFromPublic": false
}
```
- `quoteKey` = `eventId:sageId`（journal と同じ粒度）。`resonateRate` で推薦を重み付け。
- 聖典は `excludeFromPublic:true`（公開ランキング/星から除外、内部の改善示唆のみ）。

### デザイン方針
- 和の温かみ＋落ち着き。余白広め、角丸、淡い和紙テクスチャ。
- 偉人カードは色（`sages.json` の `color`）でスタンスを示す。
- スマホ縦持ち最適化（ポートフォリオ閲覧の多くがモバイル想定）。

### 6.5 音声方針 ★「生の声」の現実的な設計
ユーザー要望：**できれば本人の生の声で聞けたら、なお良い。** 体験価値は非常に高いが、人物の年代によって実現手段と権利が全く異なる。`audio.type` で扱いを分け、**権利がクリーンなものだけを本番に載せる**。

| 区分 | 対象例 | 手段（`audio.type`） | 権利・注意 |
|---|---|---|---|
| 一次録音あり | ジョブズ、マンデラ、アリ、キング牧師 | `archival`（実音声の引用） | 録音の著作権・肖像/パブリシティ権の確認が必須。許諾 or 引用法の範囲、PD音源を優先 |
| 録音が存在しない（歴史上） | 孔子、紫式部、ダ・ヴィンチ | `voiceover`（プロ声優の朗読） | 「イメージボイス」と明示。本人音声と誤認させない |
| 補完・多言語 | 全般 | `tts`（高品質TTS）/ `none` | 既存人物の声を模した合成は**使わない**（なりすまし回避）。中立ナレーション用途に限定 |

**★確定方針：許諾済み/PD録音 ＋ プロ声優のイメージボイスで正攻法。AI音声クローンは不採用。**

**原則**
- 各音声に `rights`（`public-domain` / `licensed` / `fair-use` / `original-recording`）を持たせ、出典を明記。**権利不明の音源は載せない。**
- 実在人物の声を**AIで合成・クローンしない**（パブリシティ権・倫理・課金審査リスク）。歴史上の人物は「公式イメージボイス（声優）」として正々堂々と提供する。
- 音声は任意再生（テキストだけでも完結）。データが無い場合は静かにテキスト体験へフォールバック。
- 段階導入：MVPはテキスト主体（`audio.type:"none"`許容）→ v1.5でPD/許諾済みの`archival`と看板偉人の`voiceover`を追加 → v2で拡充。

## 10. 課金品質とマネタイズ（"課金できる質"の定義）

ユーザー要望：**課金可能な質のゲームにしたい。** そのための品質基準と収益モデルを定義する。

### 10.1 課金に値する品質バー（Quality Bar）
"無料名言アプリ"と一線を画すための必達ライン。
- **データの信頼性**：全名言が出典付き・裏取り済み（`verified:true`）。誤帰属ゼロ。← 最大の差別化＆課金理由。
- **キュレーション**：「悩み別×心の状態別」で"自分に効く言葉"に最短で出会える設計。量より"効く順"。
- **世界の網羅性**：地域・時代・分野のバランス（東西/古今/思想〜スポーツ）。日本人に閉じない。
- **生の声 / イメージボイス**：権利クリーンな音声で没入感（§6.5）。
- **手触り**：演出・タイポ・余白・効果音まで作品として整っている（ポートフォリオの看板になる水準）。
- **継続価値**：コレクション、座右の偉人、日めくりなど"また開きたい"仕掛け。

### 10.2 マネタイズ・モデル（★確定：偉人フェーズから課金 ＋ 経験ゲート）
**課金の境界 ＝ 「偉人フェーズ」。** 偉人が登場する前までは無料、世界の偉人（＝最大の価値であるDB）に出会うところから課金。さらに偉人内はランク制で、高ランクは経験で解放（§4.3）。

**無料 / 課金の線引き**

| 区分 | 含むもの |
|---|---|
| **無料** | 人生シミュレーション全体：イベント・悩み・決断・ステータス変化・基本の励まし（**「灯火の声」＝ナレーター/`original`**による一般的な励まし）・コレクションの土台 |
| **課金（偉人フェーズ）** | 世界の実在偉人が相談相手として登場。出典付きの本物の名言、偉人別の助言、生の声、高ランク偉人への道 |

- **無料だけでも"ゲームとして成立し、効いた体験ができる"**ことを必達（無料が薄いと課金前に離脱）。灯火の声で「まず救われる」体験を無料で届け、「本物の偉人に相談したい」で課金へ橋渡し。
- 課金商品（買い切り）：
  - `unlock_sages`（偉人フェーズ解放・入口）
  - 偉人パック：`pack_business`（実業）/`pack_art`（芸術）/`pack_thought`（思想）等、地域・分野で束ねる
  - テーマ別名言集：`pack_love`（恋愛）/`pack_work`（仕事）/`pack_money`（お金）
  - `pack_voice_archival`（許諾済み生の声）
- **課金で高ランク偉人は直接買えない**（§4.3）。課金は"出会える土俵"を開くだけ、出会いは経験で勝ち取る → 健全＆長く遊べる。
- データを `isPremium` / `packId` / `rank` / `unlock` でMVPから構造化（決済実装は後でも、商品・進行設計は最初から）。

（不採用：全体買い切り／サブスク。将来サブスク化する場合も本モデルの上に日めくり配信を乗せる形で拡張可能。）

### 10.3 配布チャネル
- まずは**Web（このポートフォリオ作品）**で体験版を公開し反応を見る。
- 手応えがあれば PWA / ストア（App Store・Google Play）へ。ストア課金審査では §5.3・§6.5 の**権利クリーンさが必須要件**になる点に留意。

## 12. プレイ後フィードバック反映（v1.5 デザイン方針）

実機プレイのフィードバックを反映した7つの改善方針。**B（レア・没入・祝福）は実装済み**、A/C は設計確定（順次実装）。

### ① レア度システム ★実装済
名言に **並／希／神聖／伝説** の4段階を付与。導出：原則は偉人ランク（rank1-2=並、rank3-4=希、聖典rank5=神聖）、伝説は専用プール。色＋ピル表示で“格”を可視化。

### ② 進め方を複数（②）— 設計確定
タイトルで入口を二分：**「人生を歩む」（自動展開）** と **「悩みを相談する」（カテゴリを選んで今すぐ言葉をもらう＝実用）**。データ共通、入口のみ分岐。

### ③ 読み飛ばし防止 — 設計確定
強制せず“自然に読ませる”。言葉のゆっくり表示（タイプ/フェード）＋表示完了まで選択ボタン非活性、選択時に「受け取る」一拍と深呼吸の間。じっくり読む人に小さな叡智ボーナス。癒しの世界観を壊さない範囲で。

### ④ 中断・再開／やり直し — 一部実装＋設計
進行は自動保存→次回続きから（実装済）。「最初からやり直す」も可（実装済）。**集めた叡智・伝説図鑑は人生を越えて引き継ぐ＝メタ進行**（伝説図鑑は実装済：`codex` を別キー永続）。

### ⑤ 伝説の言葉 10件＋図鑑 ★実装済
出現率ごく低の超レアを **10枚**（全件 verified:true）。叡智が高いほど遭遇率UP、`minWisdom` で段階的に解放。「わが叡智の書 > 伝説 X/10」で蒐集を可視化。探す没頭・コンプ欲を喚起。

### ⑥ レア遭遇の祝福演出 ★実装済
全画面テイクオーバー：放射状の光・金粉パーティクル・言葉がゆっくり立ち上がる「✦ 伝説の言葉に出会った ✦」。神聖（聖典）は結果画面に荘厳リボン。喜び倍増＆スクショ欲。

### ⑦ バズ＋改善ループ＋悩みの継続追加 — 設計確定
- **シェア**：名言/出会いを画像カード化してSNS共有（**出典必須**＝誤帰属拡散も防止）。「今日の一言」日替わり。
- **改善ループ**：響き度（§6.7）を蓄積→効く言葉を上位提示。匿名集計は v2 バックエンド（“可動しながら改善”）。
- **悩みの継続追加**：データ駆動なので `events.json` を足すだけ。さらに**リモート取得**にすればアプリ更新なしで新しい悩み・季節・日替わりを配信＝再訪と課金の核。

### ビジュアル方針（レア度）
並＝和紙／希＝青磁／神聖＝金／伝説＝虹がかった金＋粒子。偉人カードの“格”を演出で可視化、祝福はモバイル縦全画面で没入。

## 13. 叡智の位（段位・免許状）★実装済

プレイヤー自身を格付けする「位（くらい）」。**偉人ランク（§4.3＝どの偉人に会えるか）とは別物**で、プレイヤーの到達・蒐集の総合評価＝アイデンティティ。資格・免許のように分類し、**証明書（免許状）**で可視化する。

### 叡智ポイント（生涯記録・下がらない）
**P ＝ 到達した言葉の数 × 12 ＋ 伝説 × 120 ＋ 制覇した悩みカテゴリ × 50**
