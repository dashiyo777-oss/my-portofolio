# 『叡智の灯火（ともしび）』 設計書 / ハンドオフ仕様

> 人生の岐路で、歴史上の偉人が"相談相手"になってくれる人生シミュレーションゲーム。
>
> **このドキュメントの目的**：最終版（完成版）は **Fable** が実装する。Fable はこの設計書を起点に、迷わず完成版を作れること。本書は「何を・なぜ・どう作るか」を一通り定義したハンドオフ仕様である。

## 1. コンセプト

一言で：**「迷ったとき、歴史上の偉人があなたの相談相手になってくれる人生シミュレーション。」**

- プレイヤーは自分の人生を歩み、**仕事・人間関係・恋愛・学業・お金**などの岐路で悩む。
- その瞬間に偉人（孔子、スティーブ・ジョブズ、マザー・テレサ、坂本龍馬、アインシュタイン等）が登場し、**励まし・叱咤激励・いたわり**の言葉をくれる。
- 「正解の決断」ではなく、**自分に響いた言葉を選ぶ**体験が中心。ゲームオーバーはなく、何度でも違う偉人に出会える温かい設計。

### 体験の狙い（プレイヤーに残るもの）
- その日・その悩みに効く一言を持ち帰れる（実用的な癒し）。
- 自分だけの「人生の物語」と「名言コレクション（お守り帳）」が育つ。

---

## 2. コアループ

```
①人生イベント発生（岐路 / 悩み / 絶頂）
        ↓
②偉人が最大3人「相談に乗る」ために登場
        ↓
③それぞれ違う立場から言葉をくれる（楽観 / 現実 / 情熱 など）
        ↓
④プレイヤーが響いた言葉を選び、決断する
        ↓
⑤心の状態（メンタル）と人生ステータスが変化
        ↓
⑥選んだ名言が「叡智コレクション」に永久保存
        ↓
①へ（次のイベント）
```

---

## 3. ゲーム設計の4つの軸

| 軸 | 内容 |
|---|---|
| 悩みカテゴリ | 仕事 / 人間関係 / 恋愛 / 学業 / お金 / 健康 / 自己実現 |
| 心の状態 | 落ち込み・舞い上がり・迷い・不安。状態で出る言葉が変わる |
| 偉人の個性 | 同じ悩みでもジョブズと孔子では答えが真逆 → 相性が生まれる |
| コレクション | 集めた名言を「お守り」として後で見返せる |

### 3.1 ステータス（人生パラメータ）
0〜100の連続値で保持。決断で増減する。

- `mind`（心のHP / メンタル）— 0でもゲームオーバーではなく「休息イベント」へ
- `wisdom`（叡智 / 経験値）— 名言を集める・熟考の選択で上昇
- `bonds`（絆 / 人間関係）
- `wealth`（お金）
- `passion`（情熱 / やる気）

UIには上位2〜3個のみ常時表示し、残りは詳細画面で見せる（情報過多を避ける）。

## 4. 偉人キャラクター

### 4.1 ロスター（初期8人・MVP対象）
各偉人は「立場（スタンス）」を持ち、同じ悩みに別角度で答える。

| ID | 偉人 | スタンス | 一言キャラ |
|---|---|---|---|
| `confucius` | 孔子 | 徳・自省 | 静かに本質を突く |
| `jobs` | スティーブ・ジョブズ | 情熱・直感 | 背中を押す |
| `teresa` | マザー・テレサ | 慈愛・献身 | そっと包む |
| `ryoma` | 坂本龍馬 | 行動・変革 | 豪快に励ます |
| `einstein` | アインシュタイン | 探究・好奇心 | 視点を変える |
| `nightingale` | ナイチンゲール | 自律・献身 | 凛と支える |
| `lincoln` | リンカーン | 忍耐・誠実 | 粘り強さを説く |
| `murasaki` | 紫式部 | 機微・情緒 | 心の襞に寄り添う |

> 拡張枠（v2以降）：手塚治虫、キュリー夫人、渋沢栄一、ガンジー など。データ追加のみで増やせる構造にする。

### 4.2 相性（任意機能・v1.5）
プレイヤーの選択履歴から「よく言葉を選んだ偉人」を“座右の偉人”として表示。リピート性を高める軽い仕掛け。

## 5. データ構造

完成版は**データ駆動**にする。シナリオ・偉人・名言を JSON で分離し、コードを触らず増やせるようにする。

### 5.1 偉人 `data/sages.json`
```json
[
  {
    "id": "jobs",
    "name": "スティーブ・ジョブズ",
    "era": "1955–2011",
    "stance": "passion",
    "portrait": "images/sages/jobs.png",
    "color": "#3b6ea5"
  }
]
```

### 5.2 イベント `data/events.json`
```json
[
  {
    "id": "work_failure_01",
    "category": "work",
    "mood": "down",
    "title": "仕事で大失敗。もう辞めたい…",
    "body": "任されたプロジェクトで大きなミスをしてしまった。周りの目が痛い。",
    "advices": [
      {
        "sageId": "confucius",
        "quote": "過ちて改めざる、これを過ちという。",
        "note": "失敗ではなく、まだ直していないだけだ。",
        "effects": { "mind": +5, "wisdom": +8 }
      },
      {
        "sageId": "jobs",
        "quote": "点と点は、あとで繋がる。",
        "note": "今日の失敗が、いつかお前の物語になる。",
        "effects": { "mind": +8, "passion": +6 }
      },
      {
        "sageId": "teresa",
        "quote": "大切なのは、どれだけのことをしたかではなく、どれだけ心を込めたか。",
        "note": "",
        "effects": { "mind": +10, "bonds": +4 }
      }
    ]
  }
]
```

### 5.3 名言の事実性ポリシー（重要）
- `quote` が**実在偉人の実際の言葉（または定訳）**であること。出典を `source` 任意フィールドで持てるようにする。
- 出典が曖昧／創作の励ましは、`sageId` を実在人物にせず `"original"`（ナレーター／灯火の声）として扱い、混同を避ける。
- 翻訳引用は意訳である旨を `note` で補える構造にする。

### 5.4 セーブデータ（`localStorage`）
```json
{
  "stats": { "mind": 70, "wisdom": 25, "bonds": 50, "wealth": 40, "passion": 60 },
  "collected": ["work_failure_01:jobs", "money_01:ryoma"],
  "favoriteSage": "jobs",
  "turn": 12,
  "history": [ { "eventId": "work_failure_01", "chosen": "jobs" } ]
}
```

## 6. 画面仕様（MVP）

1. **タイトル画面**：作品名 / 「はじめる」「叡智コレクションを見る」。
2. **イベント画面**：上部にステータスバー、中央にイベント文、下部に偉人カード最大3枚（顔・名前・一言）。
3. **決断演出**：選んだ言葉が大きく表示 → ステータス変化アニメ → 「コレクションに加わった」トースト。
4. **叡智コレクション画面**：集めた名言を“お守り帳”として一覧。偉人・カテゴリでフィルタ。各カードをタップで全文＋出典。
5. **休息イベント**：`mind` が低いとき発生。決断なしで回復し、いたわりの言葉だけをもらえる。

### デザイン方針
- 和の温かみ＋落ち着き。余白広め、角丸、淡い和紙テクスチャ。
- 偉人カードは色（`sages.json` の `color`）でスタンスを示す。
- スマホ縦持ち最適化（ポートフォリオ閲覧の多くがモバイル想定）。

## 7. 受け入れ基準（MVP / Definition of Done）

- [ ] タイトル → イベント → 決断 → コレクション の一周が動く。
- [ ] イベント **10件以上** × 偉人 **8人**、名言は重複なし。
- [ ] 決断でステータスが変化し、上位2〜3個が画面に反映される。
- [ ] 選んだ名言が `localStorage` に保存され、再訪時に残る。
- [ ] `mind` 低下時に休息イベントが発生する。
- [ ] スマホ縦画面で崩れない（375px幅で確認）。
- [ ] 名言の事実性ポリシー（§5.3）を満たす。
- [ ] 追加ライブラリ・ビルド工程なしで `index.html` を開くだけで動く。
