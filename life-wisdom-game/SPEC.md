## 1. コンセプト

一言で：**「迷ったとき、世界中の偉人があなたの相談相手になってくれる人生シミュレーション。」**

- プレイヤーは自分の人生を歩み、**仕事・人間関係・恋愛・学業・お金**などの岐路で悩む。
- その瞬間に**世界の**偉人・有名人・成功者（孔子、スティーブ・ジョブズ、マザー・テレサ、坂本龍馬、アインシュタイン、ネルソン・マンデラ、オードリー・ヘプバーン等）が登場し、**励まし・叱咤激励・いたわり**の言葉をくれる。
- 「正解の決断」ではなく、**自分に響いた言葉を選ぶ**体験が中心。ゲームオーバーはなく、何度でも違う偉人に出会える温かい設計。

### 体験の狙い（プレイヤーに残るもの）
- その日・その悩みに効く一言を持ち帰れる（実用的な癒し）。
- 自分だけの「人生の物語」と「名言コレクション（お守り帳）」が育つ。

### ★コアバリュー（本作の最大の価値）
**「世界中の偉人・有名人・成功者の言葉を、悩みの文脈で引ける高品質データベース」そのものが資産。**

- 日本人に閉じず、東西・古今・分野（思想・科学・芸術・スポーツ・実業・人道）を横断する。
- 単なる名言集ではなく、「**どんな悩み・心の状態に効くか**」でタグ付けされている点が差別化。一般的な名言アプリは"出典別"、本作は"悩み別×心の状態別"で引ける。
- 可能なら**生の声（本人の実音声 / 一次資料の録音）**を添え、文字を超えた体験にする（→ §6.5 音声方針）。
- この**データの質・量・正確さ・権利のクリーンさ**が、課金に値する品質（→ §11）の核になる。

```
        ↓

```

## 3. ゲーム設計の4つの軸

| 軸 | 内容 |
|---|---|
| 悩みカテゴリ | 仕事 / 人間関係 / 恋愛 / 学業 / お金 / 健康 / 自己実現 |
| 心の状態 | 落ち込み・舞い上がり・迷い・不安。状態で出る言葉が変わる |
| 偉人の個性 | 同じ悩みでもジョブズと孔子では答えが真逆 → 相性が生まれる |
| コレクション | 集めた名言を「お守り」として後で見返せる |

### 3.1 ステータス（人生パラメータ）
0〜100の連続値で保持。決断で増減する。

- `mind`（心のHP / メンタル）— 0でもゲームオーバーではなく「休息イベント」へ
- `wisdom`（叡智 / 経験値）— 名言を集める・熟考の選択で上昇
- `bonds`（絆 / 人間関係）
- `wealth`（お金）
- `passion`（情熱 / やる気）

UIには上位2〜3個のみ常時表示し、残りは詳細画面で見せる（情報過多を避ける）。

### 5.3 名言の事実性・権利ポリシー（最重要 / 課金品質の生命線）
**課金プロダクトでは「誤った引用」「権利侵害」が致命傷**になる。データベースの信頼性こそが商品価値（§1, §11）なので、以下を厳守する。

**事実性**
- `quote` は**実在偉人の実際の言葉（または定訳）**であること。原語を `quoteOriginal`、出典を `source`（演説・著作・年など）で必ず持つ。
- 出典が曖昧／創作の励ましは、`sageId` を実在人物にせず `"original"`（ナレーター／灯火の声）として扱い、混同を絶対に避ける。
- いわゆる「誤帰属の名言（misattributed quotes）」を排除するため、各 quote は出典の一次／信頼ソースで裏取りし、`verified: true` を立てたものだけを本番採用する。
- 翻訳引用は意訳である旨を `note` で補える構造にする。

**権利（テキスト）**
- 著作権保護期間内の人物の長い引用は**短い引用（引用法の範囲）**に留めるか、権利元の許諾を得る。古典・パブリックドメインを優先的に厚くする。

**権利（肖像・音声）→ §6.5 へ。**

## 6. 画面仕様（MVP）

1. **タイトル画面**：作品名 / 「はじめる」「叡智コレクションを見る」。
2. **イベント画面**：上部にステータスバー、中央にイベント文、下部に偉人カード最大3枚（顔・名前・一言）。
3. **決断演出**：選んだ言葉が大きく表示 → ステータス変化アニメ → 「コレクションに加わった」トースト。
4. **叡智コレクション画面**：集めた名言を“お守り帳”として一覧。偉人・カテゴリでフィルタ。各カードをタップで全文＋出典。
5. **休息イベント**：`mind` が低いとき発生。決断なしで回復し、いたわりの言葉だけをもらえる。

### デザイン方針
- 和の温かみ＋落ち着き。余白広め、角丸、淡い和紙テクスチャ。
- 偉人カードは色（`sages.json` の `color`）でスタンスを示す。
- スマホ縦持ち最適化（ポートフォリオ閲覧の多くがモバイル想定）。

### 6.5 音声方針 ★「生の声」の現実的な設計
ユーザー要望：**できれば本人の生の声で聞けたら、なお良い。** 体験価値は非常に高いが、人物の年代によって実現手段と権利が全く異なる。`audio.type` で扱いを分け、**権利がクリーンなものだけを本番に載せる**。

| 区分 | 対象例 | 手段（`audio.type`） | 権利・注意 |
|---|---|---|---|
| 一次録音あり | ジョブズ、マンデラ、アリ、キング牧師 | `archival`（実音声の引用） | 録音の著作権・肖像/パブリシティ権の確認が必須。許諾 or 引用法の範囲、PD音源を優先 |
| 録音が存在しない（歴史上） | 孔子、紫式部、ダ・ヴィンチ | `voiceover`（プロ声優の朗読） | 「イメージボイス」と明示。本人音声と誤認させない |
| 補完・多言語 | 全般 | `tts`（高品質TTS）/ `none` | 既存人物の声を模した合成は**使わない**（なりすまし回避）。中立ナレーション用途に限定 |

**原則**
- 各音声に `rights`（`public-domain` / `licensed` / `fair-use` / `original-recording`）を持たせ、出典を明記。**権利不明の音源は載せない。**
- 実在人物の声を**AIで合成・クローンしない**（パブリシティ権・倫理・課金審査リスク）。歴史上の人物は「公式イメージボイス（声優）」として正々堂々と提供する。
- 音声は任意再生（テキストだけでも完結）。データが無い場合は静かにテキスト体験へフォールバック。
- 段階導入：MVPはテキスト主体（`audio.type:"none"`許容）→ v1.5でPD/許諾済みの`archival`と看板偉人の`voiceover`を追加 → v2で拡充。

## 10. 課金品質とマネタイズ（"課金できる質"の定義）

ユーザー要望：**課金可能な質のゲームにしたい。** そのための品質基準と収益モデルを定義する。

### 10.1 課金に値する品質バー（Quality Bar）
"無料名言アプリ"と一線を画すための必達ライン。
- **データの信頼性**：全名言が出典付き・裏取り済み（`verified:true`）。誤帰属ゼロ。← 最大の差別化＆課金理由。
- **キュレーション**：「悩み別×心の状態別」で"自分に効く言葉"に最短で出会える設計。量より"効く順"。
- **世界の網羅性**：地域・時代・分野のバランス（東西/古今/思想〜スポーツ）。日本人に閉じない。
- **生の声 / イメージボイス**：権利クリーンな音声で没入感（§6.5）。
- **手触り**：演出・タイポ・余白・効果音まで作品として整っている（ポートフォリオの看板になる水準）。
- **継続価値**：コレクション、座右の偉人、日めくりなど"また開きたい"仕掛け。

### 10.2 マネタイズ・モデル候補（推奨順）
1. **基本無料＋コンテンツ課金（推奨）**：基本の偉人/悩みは無料、追加の「偉人パック」「テーマ別名言集（恋愛/仕事/お金）」「生の声パック」を買い切りで販売。データベースの拡張＝商品、と相性が良い。
2. **買い切りプレミアム**：完成度の高い一本を有料配布（広告なし）。シンプルだが集客が課題。
3. **サブスク**：日めくり配信・新規偉人の継続追加で月額。運用コストとコンテンツ供給力が要る。

> MVPは課金を実装しないが、**データを`isPremium`等でパック分けできる構造**にしておき、後から1のモデルに無改造で移行できるようにする。

### 10.3 配布チャネル
- まずは**Web（このポートフォリオ作品）**で体験版を公開し反応を見る。
- 手応えがあれば PWA / ストア（App Store・Google Play）へ。ストア課金審査では §5.3・§6.5 の**権利クリーンさが必須要件**になる点に留意。
