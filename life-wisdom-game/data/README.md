## 事実性ポリシー（§5.3 の運用）
各助言に `source`（出典）と `verified` を必須化。

- `verified: true` … 一次資料・定訳で出典（章節・書簡・演説名）まで裏取り済み。
- `verified: false` … **広く帰属するが一次出典の最終確認を推奨**。本番採用前に監修確認する。
  - `relationship_conflict_01` ガンジー「弱い者は赦せない…」
  - `self_doubt_01` ガンジー「強さは…不屈の意志から」
  - `self_doubt_01` モハメド・アリ「練習の一分一秒が嫌だった…」
  - `health_illness_01` マリ・キュリー「人生に怖れるべきものは何もない…」

> 方針：誤帰属の名言が氾濫する領域だからこそ、「裏取り済みだけを本番に出す」ことが課金に値する品質の核（§10.1）。`verified:false` は出荷前にゼロにするか、`original`（灯火の声）へ振り替える。

## 聖典コンテンツの扱い（§4.4）
- 神／アッラー／仏を**一人称で演じない**。`scripture_*` という聖典IDで「一節を出典付きで賜る」形。
- 引用は**章節まで明記**（例：マタイ 11:28、クルアーン 94:5–6、法句経 第5偈）。
- 中立・非布教。三大宗教（キリスト教／イスラム／仏教）を核に。神道は扱わない。
- 視覚：イスラムは神・預言者を描かない。聖典は光・カリグラフィー・象徴で表現。

## 課金境界（§10.2）
- `original`（灯火の声）= 無料。`isPremium:true` の偉人・聖典 = 偉人フェーズ（課金）。
- 高ランク偉人・聖典は課金で直接買えず、経験（`wisdom`）で解放（§4.3）。

## わが叡智の書（§6.6）
- プレイヤーの決断は `journal[]`（セーブ §5.4）に記録され、後から「経験した悩み×受けた言葉」を辿れる。
- `events.json` の各 advice の `quote`/`source` は、決断時に journal へ**スナップショット保存**される（後日データ更新でも当時の言葉が不変）。
- ゆえに `source` は常に正確であること（誤帰属は記録に焼き付き、共有時に拡散するため・§5.3）。

## 拡張方法
1. 新しい偉人は `sages.json` に追記（`rank`/`packId`/`isPremium`）。
2. 新しい悩みは `events.json` に追記。`advices[].sageId` で偉人を参照。
3. 出典のない励ましは `sageId:"original"` にする（実在人物に帰属させない）。
