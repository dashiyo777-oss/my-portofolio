# 縦型宣伝ショート動画 制作ブループリント

SNS（YouTube ショート / Instagram Reels / TikTok）向けの**縦型9:16の宣伝ショート動画**を、
コードだけで（外部の動画編集ソフトなしに）作り、**音声付きMP4として書き出す**ための型。

> 実装済みの再利用スキル: [`.claude/skills/promo-short-video/`](../.claude/skills/promo-short-video/SKILL.md)
> 実例: `eichinohi-short-3.*`（日本語）/ `eichinohi-short-en-1〜3.*`（英語）と生成器 [`scripts-gen/gen_en_shorts.py`](../scripts-gen/gen_en_shorts.py)

---

## 1. 倫理（最優先）

`professional-writing-blueprint.md` と同じ原則を、動画にもそのまま適用する。

- **嘘を売らない**。名言・引用は**一次資料に出典のある検証済みのものだけ**を使う。実在の人物に言っていないことを言わせない。
- 引用は**画面に出典を明記**する（例: 『独行道』1645 / *Letters*, 71）。英語圏向けは、原文が英語の人物は**英語原文**を使う（訳し戻さない）。
- 誇張コピーを避け、**体験の事実**（「言葉が岐路を照らす」など）だけを約束する。
- 「無料」等の訴求は事実に限る。

## 2. 構成（尺は30〜40秒）

短尺は**1動画1テーマ**。掴み→本編→CTAの型に落とす。

```
フック(3.5s) → シリーズ表示(3s) → [悩み(2.5s)→賢人の言葉(6s)]×3 → まとめ(3s) → CTA(4s)
```

- **フック**: 視聴者の状況を一言で言い当てる（「人生には、迷う夜がある。」）。冒頭2秒で離脱が決まる。
- **悩み→答えの反復**: 「問い」を投げてから「答え」を出すと、視聴者が自分事として引き込まれる。3回の反復でリズムを作る。
- **数字の戦略配置**: 「世界の偉人27人」など、具体的な数字を1つ効かせる。
- **CTA**: ブランド名＋一言価値＋URL＋「無料」を短く。海外向けは英日併記でブランドの世界観を残す。

## 3. 技術の型（HTML → 決定論レンダリング → ffmpeg）

動画編集ソフトを使わず、**1枚のHTMLがそのまま動画のソース**になる。

1. **統一エンジンのHTML**を作る。全ての見た目（炎の揺れ・火の粉・各シーンの不透明度・プログレスバー）を
   **時刻 `t`(ms) の純粋関数 `window.renderAt(t)`** で表現する。
   - 通常アクセス → `requestAnimationFrame` で実時間再生（BGM付き）。ユーザーはそのまま視聴・画面録画できる。
   - `?capture=1` → 自動再生せず、外部から `renderAt(t)` を呼ぶだけ。UI/バーは隠す。
2. **決定論フレーム書き出し**: Playwright(Chromium) で `?capture=1` を開き、
   `renderAt(i*1000/fps)` を呼んで**30fpsで1枚ずつ**スクリーンショット（JPEG）。
   → リアルタイム画面録画は使わない（後述の理由）。
3. **ffmpeg合成**: 連番JPEG → H.264(yuv420p) + AAC、BGMをフェードイン/アウトして多重化。
   `-movflags +faststart` を付け、そのままSNS投稿できるMP4にする。

## 4. この環境での落とし穴（重要）

実際に踏んだ罠と回避策。次回これで詰まらないこと。

- **リアルタイムのheadless画面録画は使えない**。起動直後に十数秒の白画面が入り、尺も伸びる（39秒→53秒等）。
  → **必ず決定論レンダリング**（`renderAt(t)` を1フレームずつ）にする。白フレーム無し・尺が正確。
- **同梱のffmpeg（Playwrightのbin）は貧弱**（VP8のみ、H.264/音声エンコーダ無し）。
  → `pip install imageio-ffmpeg` で**フル機能のffmpeg**（libx264 + aac）を入れる。`get_ffmpeg_exe()` でパス取得。
- **Google Fontsはプロキシでブロックされる**が、ブラウザのCJK/セリフ**フォールバックで綺麗に出る**ので実害なし。
- **CSSアニメーションと `renderAt` のインラインstyleは競合する**。炎・火の粉もCSS keyframesではなく `renderAt` で駆動する（両立のため keyframes は使わない）。
- Chromium binは `/opt/pw-browsers/chromium-*/chrome-linux/chrome`、`--no-sandbox` 必須。

## 5. 世界観トークン（叡智の灯火の例）

流用時は色・フォント・炎モチーフを変えるだけ。

- 背景 `#2e1c0a`（暖色ダークブラウン）/ ゴールド `#c8860a` / 琥珀 `#d4863a` / 文字 `#fdf6ec`
- フォント: 見出し・引用は Serif（Noto Serif JP / Cormorant Garamond）、ラベルは Sans（Montserrat）
- 中央に揺れる炎＋ハロー、背景に立ち上る火の粉

## 6. 書き出し前チェックリスト

- [ ] 引用はすべて検証済み・出典明記か（創作名言ゼロ）
- [ ] 1動画1テーマ／フックで状況を言い当てているか
- [ ] 尺は30〜40秒、9:16(1080×1920)か
- [ ] `renderAt(t)` で決定論書き出ししたか（白フレーム・尺ズレ無し）
- [ ] MP4は H.264+AAC・`+faststart`・BGMフェード付きか
- [ ] CTAにURL・ブランド・（事実の）無料訴求が入っているか
- [ ] 複数言語版は原文優先＋文化に合った人選か
